# 一、公众号长文：

## 标题候选

- **「我做了一个《人民的名义》版 MBTI，测完我自己笑了半天」**
- **「32 题，把 16 位汉东干部装进你的坐标系」**
- **「祁同伟型人格，真的存在吗？我写了个网站来回答」**

## 文章（约 2200 字）

---

**一、起心动念**

前阵子重刷《人民的名义》，发现一个现象：十个弹幕里有八个在「自我对号」。有人说"我身边就有一个李达康"，有人说"这不是我上一任老板嘛"，还有更离谱的——"这剧其实没讲贪污，讲的是职场的每一种悲剧都是人格悲剧"。

我想：与其让大家隔着屏幕猜，不如真做一个测评，让每个人直接把自己放到汉东省那张地图上。

于是就有了这个站：**汉东省人格档案室**。32 道题，5 分钟，匹配到剧里 16 位人物里最像的那一个。

**二、为什么不是 MBTI**

通用 MBTI 的四个维度——外向/内向、直觉/实感、思考/情感、判断/感知——在剧里用不上。**《人民的名义》的戏骨们，吵的不是这个。**

他们吵的是：
- **理想 vs 现实**：你做决定时，更看信念还是更看局面？（沙瑞金 vs 高育良）
- **规则 vs 权变**：规定和实效之间，你站哪边？（易学习 vs 李达康）
- **协作 vs 独断**：你倾向拉人进来一起做，还是自己拍板？（侯亮平 vs 祁同伟）
- **进取 vs 守成**：你更信破局的机会，还是时间的沉淀？（李达康 vs 陈岩石）

这四条才是剧里每一场饭局、每一次常委会真正的分歧线。所以测评换了坐标，匹配的是**人**不是**标签**——测完你看到的不是"你是 INTJ"，是"你是祁同伟"。

**三、16 位干部，16 种人生**

每一位角色都写了一份 700-800 字的档案长文，用第二人称。不是网文那种凑字数的"你的优点是……你的缺点是……"，而是试图写出那个人**在剧里真正的样子**。举个例子——

> "你是祁同伟型……你的起点太低，让你把'进步'两个字烫在了骨头上；你的聪明和努力是真的，不然你不会从一个农村娃走到厅级。但你心里有一片阴影：这世界曾在你最要强的时候羞辱过你，你告诉自己再也不要那样，于是你给自己立了一个规矩——**只要能往上走，代价我来扛**。"

对——他那句著名的「**我太想进步了**」，原样保留在了他的档案里。

其他角色的金句：侯亮平的"人民的名义，不容亵渎"；沙瑞金的"党的旗帜不是用来抖的"（改编）；李达康的"GDP 是干出来的不是吹出来的"（剧中概念）；孙连城的……就不剧透了，测出来你就知道了。

**四、你的档案里还有什么**

除了匹配到的那一位，你还会看到：
- **四维度分数条**：你在每一条光谱上的偏移度
- **最相近的另一个人**：和你第二像的是谁
- **最相远的那一个**：以及剧中谁和你最拧巴
- **所属阵营**：沙李派 / 汉东帮 / 京州资本 / 中立

最后一张一键生成的**竖版分享长图**，1080×1920，带二维码，朋友扫码就能进到你同一份档案页。

**五、我们没做的事**

这个站**没有后端，没有数据库，没有账户系统**。你的答题结果完全编码在 URL 的 `#` 之后——这意味着：

- 数据永远不会离开你的浏览器
- 关了标签页，档案就没了
- 但你复制链接给朋友，朋友打开看到的是**同一份档案**
- 我们既看不到你的测评结果，也不知道你是谁

这就是粉丝项目最舒服的姿态：做一个东西给大家玩，不做生意。

**六、一点免责**

本站是《人民的名义》非商业粉丝项目，**与剧集出品方无任何关联**。所有角色剧照仅用于对话语境下的人物识别，若有任何形式的下架要求，随时下架。"汉东省人格档案室"是致敬剧中虚构的汉东省，与任何真实机构无关。

**七、传送门**

[链接放这里]

测完，欢迎评论区交卷——你是汉东省的哪一号？

# 二、掘金/技术博客

## 标题候选

- **「用 Vanilla TS + Vite 做一个纯静态的 MBTI 测评站——零后端，结果跨设备可复活」**
- **「小项目，真工艺：一个《人民的名义》测评站的工程笔记」**
- **「如何把 16 人格测评的全部状态编码进 URL？」**

## 文章（约 1800 字）

---

**1. 需求**

朋友想做一个《人民的名义》主题的 MBTI 测评站，核心功能：
- 32 道李克特五档题，得四个维度的分数
- 把分数映射到 16 位剧中人物
- 出一份人格报告（1000 字级的长文 + 金句 + 最近最远邻居）
- 能分享结果，跨设备可复活

约束：**不搞后端**，一个人维护不起；**不要用户登录**，朋友圈里的人不愿意注册帐号。

**2. 架构**

```
vite + TypeScript strict + bun + oxlint
无框架，原生 DOM + hash router
@chenglou/pretext 做中文长文精排
qrcode-generator 画分享图二维码
sips 处理剧照
部署：Cloudflare Pages（纯静态）
```

整个前端 `dist/` 打包后 gzip 47 KB。

**3. 状态全部塞进 URL**

既然不要后端，结果状态必须编码进 URL，才能做到「复制链接给朋友，朋友打开看到的是同一份档案」。

编码方案：
- 每道题的答案是 `-2 | -1 | 0 | 1 | 2`，共 5 种 → 3 bit 刚好
- 32 题 × 3 bit = 96 bit = 12 byte
- 用 **base32hex** 编码为 20 字符
- 前缀 `v1.` 留升级口子

```ts
// src/engine/encode.ts
export function encodePayload(answers: Answer[]): string {
  // 3 bits per answer, pack into 12 bytes, base32hex encode
  // ...
}
```

完整 URL 长这样：`https://xxx/#/r/v1.A1B2C3...`

**4. 引擎是纯函数**

`src/engine/` 里所有函数都是纯函数：`computeScores(answers) → Scores`、`classify(scores) → {typeCode, character}`、`nearestAndFarthest(scores, self) → {nearest, farthest}`。

好处：
- 单测 27 个 all pass，覆盖所有边界（低强度阈值 / 全 0 分 / 反向题）
- 后续如果要加题，只要改 `questions.ts`
- 前端也不需要 Redux 之类的状态容器

**5. 16 人格的映射**

4 个维度，每维度二元，组合是 `2^4 = 16`，正好对上 16 位主要角色。

比如 `RLAD`（现实主义 · 规则坚守 · 独断专行 · 锐意进取）→ 李达康。

一个边界情况：如果用户所有四个维度分数都 `|score| ≤ 2`（低强度），就不能硬套——所以落到一个"没脾气"的角色：**孙连城**（那个看星星的区长），对应 `RLCP`。

**6. 中文长文的精排**

单人长文 700-800 字，如果直接丢给浏览器，CJK 断行控制力很差（首尾标点、孤字、弱断点全靠运气）。用了 **chenglou/pretext** 做精排——

```ts
import { prepareWithSegments, layoutWithLines } from '@chenglou/pretext';

const prepared = prepareWithSegments(text, '17px/1.8 "Songti SC", serif');
const { lines } = layoutWithLines(prepared, maxWidth, lineHeight);
// lines: Array<{ text, width, start, end }>
```

然后每行用一个 `<div class="pretext-line">` 单独渲染。首行缩进、禁首尾标点全部控住。

**坑**：pretext 靠 canvas 测量，所以必须在 `document.fonts.ready` 之后跑；段落还要按源文的 `\n\n` 切了单独渲染，不然视觉上全黏一起。

**7. 分享长图**

1080×1920 的竖版 PNG，用 `<canvas>` 合成：

```ts
const canvas = document.createElement('canvas');
canvas.width = 1080 * 2; // DPR 2
canvas.height = 1920 * 2;
const ctx = canvas.getContext('2d');
ctx.scale(2, 2);
// 绘制：报头、照片框、信息表、金句、四维条、二维码、"已归档"红章
canvas.toBlob(blob => { /* a.download */ });
```

二维码用 `qrcode-generator`（极简库，20 KB 不到），直接在 canvas 上按 module 绘制，不需要 DOM。

**8. 视觉**

档案卷宗风：宋体系统栈、朱砂红 `#8b2e2e`、牛皮纸 `#f4ece0`、双重下划线、旋转的红章、虚线描边。字体没打自托管 woff2——系统宋体栈（`Songti SC` / `Noto Serif CJK SC` / `STSong`）够用，节省一大块包体。

## 1. 项目概述

一个致敬电视剧《人民的名义》的人格测评网站。用户回答 32 道情境题后，系统在 4 个政治哲学维度上给出分数，匹配到 16 位剧中人物之一，并给出阵营归属与人格长文，支持长图 + 二维码的分享形式。

### 1.1 设计哲学
- **档案卷宗美学**：页面模拟"干部人格评估档案"，牛皮纸底 / 朱砂印章 / 宋体 / 折痕阴影。
- **工程极简**：参考 `chenglou/pretext` 的工程风格（Bun + TypeScript + 极少依赖 + 无框架）。
- **纯静态**：结果编码在 URL hash 中，无后端、无数据库、无账户。零运维。
- **文字精排**：人格长文使用 `@chenglou/pretext` 精确布局（断行、禁首尾标点、中文长破折号矫正）。

### 1.2 非目标（Non-goals）
- 不做用户系统 / 登录 / 保存历史
- 不做全站统计 / 排行榜 / 热度图
- 不做小程序 / 原生 App
- 不做多语言

## 2. 测评模型

### 2.1 四维度

| 代码 | 名称 | 正向含义 | 反向含义 |
|---|---|---|---|
| 第 1 维 | 价值观 | **I** 理想主义 | **R** 现实主义 |
| 第 2 维 | 规则观 | **L** 规则坚守 | **F** 权变灵活 |
| 第 3 维 | 协作观 | **C** 集体协作 | **A** 独断专行 |
| 第 4 维 | 节奏观 | **D** 锐意进取 | **P** 谨慎守成 |

- 每维度 8 题，共 32 题。每题 5 档李克特量表，映射到 [-2, -1, 0, +1, +2]。
- 正反问题各半（每维 4 正 4 反），反向题自动取反。
- 维度总分 ∈ [-16, +16]。
- 维度代码取正负：`≥ 0` → 字母前项（I/L/C/D），`< 0` → 字母后项（R/F/A/P）。

**低强度特判**：若所有四维 `|score| ≤ 2`（合计表现几乎中立），不按 16 型映射，直接匹配到**孙连城**（对应 RLCP + 佛系中立）。目的是避免"全中立用户"被机械地落到某一具体强角色身上。

### 2.3 阵营归属（由匹配角色查表得出）

阵营是"你在剧情中会站哪边"的定性标签，**直接由 2.2 匹配到的角色查下表**得出，不额外引入分数判定，避免角色与阵营相互冲突。

| 阵营 | 角色 |
|---|---|
| 沙李派（改革阵营） | 侯亮平、沙瑞金、陈岩石、陈海、易学习、钟小艾、季昌明、田国富、李达康、郑西坡 |

| 佛系中立 | 孙连城 |

实现在 `engine/faction.ts` 中是一个 `Record<CharacterKey, Faction>` 查表常量 + 简单的 `factionOf(character)` 函数。

### 2.4 最相近 / 最相远角色

基于 4 维欧氏距离，在 16 人理想分数中心（每维 ±16）上找 Top1（最相近）和 Bottom1（最相远）。结果页展示两个次级角色缩略名片，增加趣味性。

## 3. 体验流程

```
首页 /                    （档案封面）
  → [开始测评]
答题页 /#/q               （32 题，进度条）
  → 提交
结果页 /#/r/<payload>      （URL hash 编码的分数）
  ├─ 档案照（剧照 + 姓名/职务样式）
  ├─ 类型码 + 金句
  ├─ 四维刻度条
  ├─ 人格长文（pretext 精排，~1200 字）
  ├─ 阵营徽章
  ├─ 最相近 / 最相远角色
  ├─ [生成分享长图] / [复制链接]
  └─ [再测一次]
```

### 3.1 URL Hash 编码

**格式**：`#/r/<base32(answers_packed)>`

- `answers_packed`：32 题答案，每题占 3 bit（值域 0..4 对应 -2..+2），共 96 bit = 12 字节
- 编码为 base32hex（20 字符）附在 hash 后
- 结果页 cold load 时从 hash 解码 → 重算维度分 → 渲染完整档案
- 分享链接即当前完整 URL，任何人点开都能复活同一结果

**不在 URL 中存**：类型码、角色、阵营（客户端每次根据分数现算，避免 URL 变长与不一致）

**版本容错**：hash 格式 `#/r/v1.<base32>`，v1 前缀便于将来升级。

## 4. 技术架构

### 4.1 技术栈

- **运行时**：Bun（开发 + 测试）
- **语言**：TypeScript（严格模式）
- **框架**：无（原生 TS + 手写极简 DOM 辅助）
- **文字排版**：`@chenglou/pretext`
- **Lint**：oxlint
- **构建**：Bun 自带打包 或 Vite（打包静态产物）
- **部署**：Cloudflare Pages（仅静态）

### 4.3 模块边界

- **engine/\***：纯函数，0 依赖，完全可单元测试。
- **dom/\***：DOM 操作辅助，不涉业务逻辑。

- **share/\***：离屏 canvas 绘制 + 二维码，独立可测。

## 5. 视觉与内容

### 5.1 视觉系统

- **字体**：
  - 标题：思源宋体 Heavy
  - 正文：LXGW WenKai 或 思源宋体 Regular
  - 数字 / 类型码：等宽衬线（Source Serif 4）
  - 字体通过 `font-display: swap` 子集化加载，首屏不阻塞
- **色板**：
  - 背景 `#f4ece0`（牛皮纸）
  - 主文字 `#1a1a1a`
  - 印章红 `#8b2e2e`
  - 档案墨绿 `#2d4a3e`
  - 折痕阴影 `rgba(0,0,0,.08)`
  - 辅助灰 `#7a7067`
- **元素**：
  - 首页：档案袋封皮 + 装订线 + 钢印"绝密"效果
  - 答题：笔录稿纸（横线稿纸背景 + 题号手写体）
  - 结果：干部履历表版式（照片框 + 表格栏 + "阅"字红章）
  - 全站保留轻微纸张噪点纹理（SVG filter）
- **响应式**：移动端优先（大多数用户从社交平台点进来），断点 640 / 1024。

### 5.2 题库（32 题）

- 每题 20-40 字，场景化描述（"面对同事越级汇报，你更倾向于…"）
- 不暴露所属维度
- 每维度 4 正 4 反，确保反作弊稳定性
- 题目乱序呈现，但答题 state 按原始 index 记录（encode 依赖原始顺序）

### 5.3 16 人档案长文

每人约 1000-1500 字，结构：

1. **定性金句**（10-15 字，印章样式展示）
2. **人格速写**（150-250 字，直接对"你"说话）
3. **典型行为**（剧情引用 3 条，注明集数）
4. **高光时刻**（一段）
5. **阴影面 / 盲点**（一段）
6. **与你相处的建议**（给"别人如何和你相处"）
7. **最相近 / 最相远角色**（自动根据测试者的分数填充，不在静态数据中）

祁同伟一档金句固定为"**我太想进步了**"（网络名梗）+ 备用"胜天半子"（剧中原句）。

## 7. 反作弊与边界

- 正反问题自动翻转，单纯"全选同意"得到中性结果
- 低强度用户（所有维度 `|score| ≤ 2`）→ 由 2.1 的特判直接匹配**孙连城**
- URL hash 解码失败 / 越界 → 跳回首页并 toast 提示
- 分享链接被篡改：因为客户端重算，篡改只会得到另一个合法结果，不会崩溃

## 8. 测试策略

- `tests/engine.test.ts`：
  - 每个 16 型至少 1 组典型 answers → 确认落到对应角色
  - 反向题翻转正确
  - 低强度特判：所有维度 `|score| ≤ 2` → 孙连城
  - 边界：刚好超出特判阈值（`|score| = 3`）→ 进入 16 型映射
  - 类型码边界：维度恰为 0 → 映射到 R/F/A/P 端（`≥ 0 → I/L/C/D` 但 `0` 归正向端的规则）

  说明：2.1 的规则是 `≥ 0 → I/L/C/D`，所以维度为 0 时取 I/L/C/D（理想/规则/协作/进取端）。测试要显式覆盖这个语义以免回归。
- `tests/encode.test.ts`：
  - 编解码往返
  - v1 版本号前缀
  - 坏输入拒绝
- 不覆盖：DOM 渲染、canvas 长图（视觉测试手工过）

## 9. 工程化与部署

### 9.1 脚本

- `bun run dev`：本地静态服务（Bun 自带 / Vite）
- `bun test`：跑 engine + encode 单测
- `bun run lint`：oxlint
- `bun run build`：产出 `dist/`
- `bun run deploy`：`wrangler pages deploy dist`

### 9.2 Git Hook（pre-commit）

- `bun test` + `oxlint` 必过

### 9.3 分阶段交付

1. **M1 骨架**：路由 + engine + encode + 单测全绿（无视觉）
2. **M2 内容**：32 题题库 + 16 人档案文案定稿
3. **M3 视觉**：档案卷宗风全部视觉落地（首页 / 答题 / 结果）
4. **M4 分享**：长图 + 二维码
5. **M5 部署**：Cloudflare Pages 上线 + README

每个里程碑结束后做一次手动 QA + 视觉过稿。
